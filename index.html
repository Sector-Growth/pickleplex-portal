What's included — Phase 1

Everything in scope for the build.

Five deliverables that turn adding a location from an engineering task into something your own team does in a few minutes.

Your own portal

A guided onboarding app

A dedicated web application — kept separate from the sync engine — that walks your team through adding a location, one field at a time.

Config-driven

Add locations without code

A one-time change moves the location list into a configuration file the portal manages — so every future club is a form entry, never a code change.

One place

Every detail captured

Location name, CourtReserve organization ID and credentials, and the contact owner — chosen from a live list of your HubSpot users.

Git-versioned

One submit writes the setup

Submitting saves the location's configuration — targets, owner and team mapping — into the sync's source of truth, versioned and auditable.

Reuses your pipeline

The pipeline does the rest

The portal automatically updates your HubSpot properties and runs the first data sync through your existing, proven pipeline — nothing rebuilt, nothing duplicated. The same engine that runs every night simply picks up the new club.
